# Отладка: у пользователей не появляются задачи из Aspro

Задачи показываются только если:
1. У пользователя в БД заполнен **aspro_id** (ID из Aspro Cloud).
2. В Aspro у задачи в поле **Ответственный (responsible)** назначен именно этот пользователь.

---

## 1. Проверить aspro_id

**В БД (на VPS):**
```bash
sudo -u postgres psql -d team_tracker -c "SELECT id, email, aspro_id FROM users;"
```

Если у кого-то **aspro_id = NULL** — при регистрации пользователь не нашёлся в Aspro (по email). Решения:
- Пусть пользователь зайдёт в приложение → Профиль → нажмите **«Привязать к Aspro»** (если такая кнопка есть), или вызовите перепривязку (см. ниже).
- Либо вручную обновите в БД после того как узнаете aspro_id из портала Aspro (кабинет пользователей, id = aspro_id).

**Перепривязка по API (пользователь залогинен):**
```http
POST /auth/sync-aspro
Authorization: Bearer <токен пользователя>
```
В ответе будет `aspro_id` и сообщение: нашёлся ли пользователь в Aspro по email. После успешной перепривязки пользователь обновляет страницу — задачи подтягиваются.

---

## 2. Отладка списка задач для пользователя

Пусть пользователь залогинится и откроет в браузере (или через curl с его токеном):

```text
GET /tasks/raw
Authorization: Bearer <токен>
```

В ответе:
- **user_aspro_id** — какой aspro_id у пользователя (должен совпадать с БД).
- **raw_count** — сколько задач вернул Aspro до фильтров.
- **after_filter_count** — сколько осталось после фильтра (активные + ответственный = этот пользователь).
- **tasks[]** — по каждой задаче поля **passed_responsible**, **passed_active**, **in_result** и **responsible_id**.

Если **raw_count = 0** — Aspro не отдаёт задачи по этому пользователю (проверьте API-ключ, фильтры, что в Aspro у пользователя есть задачи где он ответственный).  
Если **raw_count > 0**, но **after_filter_count = 0** — смотрите по задачам: **passed_responsible: false** значит в задаче `responsible_id` не совпадает с aspro_id пользователя (в Aspro нужно назначить этого пользователя ответственным).

---

## 3. Совпадение email с Aspro

aspro_id ищется по **email**: в Aspro в списке пользователей (module/core/user/list) сравнивается поле **username** с email из нашего приложения. Если в Aspro логин другой (не email) — пользователь не найдётся, aspro_id будет NULL. Нужно либо привести логин в Aspro к email, либо завести отдельное сопоставление (например, ручное обновление aspro_id в БД).

---

## 4. Логи на сервере

При запросе задач в логах PM2 видны URL запросов к Aspro и количество задач:
```bash
pm2 logs team-tracker
```
Ищите строки вида `Aspro tasks list:`, `findAsproUserIdByEmail`, `sync-aspro`.
