# Сводная таблица: статусы задач Aspro ↔ приложение

Используется для дебага выставления статуса при выборе задачи. Сейчас в приложении этап «В работе» отображается как «Правки» — ниже цепочка маппингов.

---

## 1. Источник: Aspro API

Задача приходит с полями (варианты имён из разных эндпоинтов):

| Поле в ответе Aspro | Тип | Описание |
|---------------------|-----|----------|
| `workflow_stage_id` | number | ID этапа в workflow |
| `status` | number | Код статуса (в доке: 1 — новая, 3 — в работе, 4 — на проверке, 5 — закрыта) |
| `stage.name` / `workflow_stage.name` / `statusTitle` / `stageName` | string | Название этапа **как в Aspro** (например «В работе», «Правки», «Новая» и т.д.) |

**Важно:** реальное название этапа в вашем портале Aspro может быть любым (в т.ч. «Правки»). Список этапов workflow: `GET /module/task/stages/list`.

---

## 2. Сервер: нормализация в три статуса

**Файл:** `server/routes/tasks.js`

### 2.1 По коду статуса (если пришёл только id, без названия)

| Код (status / workflow_stage_id) | Нормализованный statusTitle |
|----------------------------------|-----------------------------|
| 0 | Не выполняется |
| 1 | Не выполняется |
| 2 | В работе |
| 3 | В работе |
| 4 | Ожидает контроля |
| 5 | Не выполняется |

Константа: `STATUS_CODE_TO_TITLE`.

### 2.2 По названию этапа из API (lowercase)

| Название из Aspro (нижний регистр) | Нормализованный statusTitle |
|------------------------------------|-----------------------------|
| не выполняется | Не выполняется |
| в работе | В работе |
| ожидает контроля | Ожидает контроля |
| новая / новый / не начато | Не выполняется |
| **правки** | **В работе** |
| на проверке / ревью / review | Ожидает контроля |

Константа: `STATUS_NAME_TO_TITLE`.

В ответе API приложения (`GET /tasks`) в каждой задаче поле **`statusTitle`** — одно из: **«Не выполняется»**, **«В работе»**, **«Ожидает контроля»** (или сырое название/«Статус N»/«Без статуса»).

---

## 3. Клиент: что видит пользователь

**Файл:** `app/src/components/TaskSelectModal.js`

Маппинг **нормализованного** statusTitle → подпись в UI:

| statusTitle (от API приложения) | Label в приложении (что видит пользователь) |
|---------------------------------|--------------------------------------------|
| Не выполняется | Не начато |
| **В работе** | **Правки** ← здесь «В работе» подменено на «Правки» |
| Ожидает контроля | На проверке |

Константа: `STATUS_FILTERS` (поле `id` = statusTitle, `label` = отображаемый текст).

---

## 4. Выставление статуса при выборе задачи

**Файл:** `server/routes/tasks.js`  
**Роут:** `POST /tasks/:id/start` (вызывается при выборе задачи в приложении).

| Шаг | Что происходит |
|-----|----------------|
| 1 | Берётся задача по `taskId` из Aspro (`getAsproTaskById`). |
| 2 | Читается `workflow_id` задачи. |
| 3 | Запрашивается список этапов: `getAsproStagesList()` → `GET /module/task/stages/list`. |
| 4 | Ищется этап с **именем строго «в работе»** (lowercase): `s.name.trim().toLowerCase() === 'в работе'`. |
| 5 | Задаче выставляется `workflow_stage_id` этого этапа через `updateAsproTask(taskId, { workflow_stage_id, workflow_id })`. |

Константа на сервере: **`STAGE_NAME_IN_PROGRESS = 'в работе'`**.

Если в Aspro этап называется **«Правки»**, а не «В работе», то POST `/tasks/:id/start` **не найдёт** этап (и вернёт 502 «Этап «В работе» не найден»), либо в приложении будет путаница: в списке этапов в Aspro — «Правки», а мы ищем «В работе».

---

## 5. Сводная цепочка «В работе» ↔ «Правки»

| Место | Значение | Комментарий |
|-------|----------|-------------|
| Aspro (название этапа в workflow) | ? | Может быть «В работе» или «Правки» — см. stages/list. |
| Сервер: STATUS_NAME_TO_TITLE | «правки» → «В работе» | Название «Правки» из API нормализуется в statusTitle «В работе». |
| Сервер → клиент: statusTitle | «В работе» | В JSON задачи уходит уже нормализованный статус. |
| Клиент: STATUS_FILTERS | id «В работе» → label «Правки» | В UI показывается «Правки» вместо «В работе». |
| POST /tasks/:id/start | Ищем этап с именем «в работе» | Если в Aspro этап называется «Правки», по имени не найдётся. |

---

## 6. Что поправить (на выбор)

1. **Только UI:** чтобы везде было «В работе», а не «Правки» — в `TaskSelectModal.js` заменить `label: 'Правки'` на `label: 'В работе'` для id `'В работе'`.
2. **Логика «В работе» на бэкенде:** если в Aspro этап реально называется «Правки», в `tasks.js` добавить в поиск этапа альтернативное имя, например считать этап «в работе», если имя «в работе» **или** «правки» (чтобы POST `/tasks/:id/start` находил этап).
3. **Единый маппинг:** вынести все пары «название в Aspro ↔ statusTitle ↔ label в UI» в один конфиг (сервер или env), чтобы не дублировать «Правки»/«В работе» в двух местах.

После правок можно свериться с этой таблицей и при необходимости обновить её.
